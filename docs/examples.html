Minimal RPO Examples

Reference examples to help implementers understand how an RPO bundle is structured, and how JSON, PDF and hashes work together in practice.

1. Minimal JSON bundle

The simplest valid RPO bundle contains only the required top-level fields:

  • rpo_version
  • bundle_id
  • created_at
  • issuer
  • subject
  • evidence[]
  • narrative
  • registry
  • signatures[]
