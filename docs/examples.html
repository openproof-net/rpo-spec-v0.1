RPO Examples • v0.1 Draft

Minimal RPO Examples for Implementers

These examples show how a valid RPO bundle is structured, how JSON and PDF interact, and how hashes are used to bind narrative and evidence together.

Minimal bundle

1. Minimal JSON bundle

The simplest valid RPO bundle contains only the required top-level fields:

  • rpo_version
  • bundle_id
  • created_at
  • issuer
  • subject
  • evidence[]
  • narrative
  • registry
  • signatures[]

Download the minimal JSON example: rpo.json

2. Human-readable PDF placeholder

The example includes a PDF placeholder file showing how the narrative report is linked to the JSON via the narrative.pdf_hash field.

Download the placeholder: rpo.pdf.txt

Repository structure

3. Full example folder on GitHub

All minimal examples are available in the repository under:

/examples/example-minimal/

This folder is the reference layout for a small RPO project: a JSON bundle, an associated narrative PDF, and any auxiliary documentation files.

Next steps

4. Next steps for implementers

Use the minimal example as a template for your own RPO engine:

  • Validate your JSON bundles against the official RPO schema.
  • Run hash integrity tests on JSON and PDF.
  • Wire your narrative generator to produce a hashed PDF.
  • Extend the structure with your own evidence fields and registries.

See also the RPO Test Suite for schema and hash validation examples.
