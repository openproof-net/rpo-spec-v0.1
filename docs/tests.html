RPO Test Suite for Implementers

A minimal validation suite for RPO engines: JSON schema checks, hash integrity verification, and PDF narrative linkage. These tests are examples, not a formal certification harness.

Core tests

1. Schema validation test

Validates candidate bundles against rpo-schema.json.

  • Ensures required fields exist and are not empty.
  • Checks types and constraints for each section.
  • Verifies structural coherence of the JSON bundle.

View test file: schema-validation.test.js →

2. Hash integrity test

Ensures that the canonical JSON produces a stable, verifiable hash for the bundle.

  • Canonicalization of the JSON bundle prior to hashing.
  • SHA3-512 digest generation.
  • Comparison with the expected public_hash anchor.

View test file: hash-integrity.test.js →

3. PDF output test

Ensures that narrative PDF content corresponds to the narrative.pdf_hash field in the bundle.

  • PDF → text extraction (canonical representation of the narrative).
  • Hashing of the narrative output using the same algorithm.
  • Matching the result with narrative.pdf_hash in rpo.json.

View test file: pdf-output.test.js →

How to use these tests

4. Using the suite in your pipeline

These tests are intentionally small. They are meant to be copied, adapted, and extended inside your own CI pipeline.

  • Start from the reference tests in the repository.
  • Add project-specific checks (custom evidence fields, registries, signatures).
  • Run the tests on every bundle before anchoring it publicly.
  • Share your own improvements via pull requests if they may help other implementers.

The goal is not to enforce a single implementation, but to make coherence and integrity testable in a reproducible way.
