RPO Test Suite

Minimal validation suite for RPO implementers: JSON schema validation, hashing integrity, and PDF narrative linkage. These tests are examples, not a mandatory certification harness.

1. Schema Validation Test

Validates candidate bundles against rpo-schema.json.

  • Ensures required fields exist.
  • Checks types and constraints for each section.
  • Verifies structural coherence of the JSON bundle.

View test file: schema-validation.test.js →

2. Hash Integrity Test

Ensures that the canonical JSON produces a stable, verifiable hash.

  • Canonicalization process for the JSON bundle.
  • SHA3-512 digest generation.
  • Comparison with the expected hash anchor.

View test file: hash-integrity.test.js →

3. PDF Output Test

Ensures that narrative PDF content corresponds to the narrative_pdf_hash in the bundle.

  • PDF → text extraction.
  • Canonical hashing of the narrative output.
  • Matching with the JSON field in rpo.json.

View test file: pdf-output.test.js →
