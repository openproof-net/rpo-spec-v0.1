Seal and public hash

An RPO bundle may include a seal section containing a deterministic public_hash. This hash is computed from canonical narrative fields and allows later verification that the content has not been altered.

This mechanism enables a narrative to be structured, sealed and later re-evaluated without rewriting history. The presence of a public hash does not imply legal registration, certification or evidentiary value by itself. 

{
  "seal": {
    "public_hash": "sha256:…",
    "method": "sha256",
    "scope": "rpo_version|bundle_id|issuer|subject|narrative",
    "note": "Deterministic seal for integrity verification"
  }
}
