OpenProof RPO Specification ‚Äî JSON + PDF proof bundle

The RPO (Rapport Probatoire Ouvert) defines a dual-format standard for digital evidence: a signed JSON bundle for machines, and a coherent narrative PDF for humans, both anchored by a public hash.

RPO v0.1 ‚Äî Minimal Guarantee
Any bundle claiming RPO compliance must implement these three pillars.
  • Signed JSON ‚Äî complete, structured evidence bundle with canonical hashing and signatures.
  • Human-readable PDF ‚Äî narrative report linked via narrative.pdf_hash.
  • Public hash ‚Äî registry anchor enabling independent verification of integrity over time.
Core components

RPO Object ‚Äî JSON Schema

The RPO JSON object is the canonical representation of the proof bundle. It defines the fields that every implementation must expose and validate.

  • Top-level fields such as rpo_version, bundle_id, created_at, issuer, subject, evidence[], narrative, registry, and signatures[].
  • Types and constraints for each field, including required vs optional sections.
  • Validation rules that allow third parties to check structural coherence of a given RPO bundle.

RPO JSON + PDF Format

The format specification describes how JSON and PDF work together as a dual-format bundle, and how they relate to hashing and registry anchoring.

  • Field-by-field description of the JSON structure.
  • Relationship between narrative sections and the underlying evidence entries.
  • Minimal rules for registry anchoring using public_hash and bundle_id.
